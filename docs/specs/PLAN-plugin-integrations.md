# PLAN: Plugin Integrations (Obsidian & Atlas Hub)

This plan details the task-by-task execution steps to implement the Obsidian knowledge bridge and Atlas state integrations in **agy-cli**.

**Goal:** Create in-process plugins for Obsidian (`agy-obs`) and Atlas (`agy-atlas-hub`) to enable notes graph analysis and session state tracking.

**Tech Stack:** Python 3.9+, SQLite3, PyYAML, Click, Rich.

---

## File Structure

| File | Status | Responsibility |
|---|---|---|
| `src/agy/plugins/obsidian.py` | Create | Database connector and query executor for Obsidian vaults. |
| `src/agy/plugins/atlas.py` | Create | Reader and parser for Atlas state registries and active sessions. |
| `src/agy/cli.py` | Modify | Integrates `obs` and `atlas` command groups to the Click CLI. |
| `tests/test_obsidian_plugin.py` | Create | Unit tests for vault graph queries and duplicate detection. |
| `tests/test_atlas_plugin.py` | Create | Unit tests for reading and parsing Atlas YAML configuration. |

---

## 🛠️ Implementation Steps

### Task 1: `agy-obs` Obsidian Database Bridge
Connect to the SQLite database generated by `obsidian-cli-ops` and execute graph queries.

*   [x] **Step 1.1: Implement Obsidian DB Connector in `src/agy/plugins/obsidian.py`**
    *   Create `ObsidianBridge` class.
    *   Read database path from `obsidian-cli-ops` config or defaults.
    *   Implement query executors for:
        *   **Orphan Notes:** Query notes with in-degree = 0 and out-degree = 0.
        *   **Hub Notes:** Query notes with high out-degree or PageRank metrics.
        *   **Broken Links:** Query links pointing to non-existent target notes.
*   [x] **Step 1.2: Add CLI Commands in `src/agy/cli.py`**
    *   Add `@click.group(name="obs")` command group.
    *   Implement subcommands: `agy obs orphans`, `agy obs hubs`, `agy obs health`.
*   [x] **Step 1.3: Add Obsidian Bridge Unit Tests**
    *   Create `tests/test_obsidian_plugin.py` with mock SQLite database fixtures and assert correct query results.

### Task 2: `agy-atlas-hub` State Synchronizer
Parse the Atlas YAML registries to extract active session metrics.

*   [x] **Step 2.1: Implement Atlas Registry Reader in `src/agy/plugins/atlas.py`**
    *   Create `AtlasBridge` class.
    *   Parse `~/.atlas/sessions.yaml` and `~/.atlas/registry.yaml` using PyYAML.
    *   Implement helper methods to extract:
        *   Active session name, duration, and context description.
        *   Recent breadcrumbs and captured inbox items.
*   [x] **Step 2.2: Add CLI Commands in `src/agy/cli.py`**
    *   Add `@click.group(name="atlas")` command group.
    *   Implement subcommands: `agy atlas status` (pretty prints active session panel) and `agy atlas trail` (displays breadcrumbs).
*   [x] **Step 2.3: Add Atlas Bridge Unit Tests**
    *   Create `tests/test_atlas_plugin.py` mocking the existence of `sessions.yaml` and asserting correct parsing.

### Task 3: CLI Testing & Verification
Verify the click subcommand routing and Rich UI presentations.

*   [x] **Step 3.1: Add CLI integration tests**
    *   Update `tests/test_cli.py` to cover `obs` and `atlas` groups.
*   [x] **Step 3.2: Run pytest validation**
    *   Execute `uv run pytest` to ensure all tests pass.
